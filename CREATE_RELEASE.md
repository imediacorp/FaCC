# Creating a GitHub Release

## Steps to Create Release v0.1.0

### Option 1: Using GitHub Web Interface

1. **Go to the Releases page:**
   - Navigate to: https://github.com/imediacorp/FaCC/releases
   - Click "Draft a new release"

2. **Fill in the release details:**
   - **Tag version:** `v0.1.0`
   - **Release title:** `v0.1.0 - Initial Release`
   - **Description:** Copy the content from `RELEASE_NOTES_v0.1.0.md`

3. **Attach release assets (optional):**
   - Source code (zip)
   - Source code (tar.gz)
   - These are auto-generated by GitHub

4. **Publish:**
   - Check "Set as the latest release"
   - Click "Publish release"

### Option 2: Using GitHub CLI

If you have GitHub CLI installed:

```bash
# Create the release
gh release create v0.1.0 \
  --title "v0.1.0 - Initial Release" \
  --notes-file RELEASE_NOTES_v0.1.0.md \
  --latest
```

### Option 3: Using Git Tags

```bash
# Create and push the tag
git tag -a v0.1.0 -m "Release v0.1.0 - Initial Release"
git push origin v0.1.0

# Then go to GitHub and create release from the tag
```

## What Happens After Release

The GitHub Actions workflow will automatically:
- Run tests across Python 3.9, 3.10, 3.11, and 3.12
- Perform linting checks
- Run basic functionality tests
- Attempt to build source distribution (if configured)

You can monitor the workflow at: https://github.com/imediacorp/FaCC/actions
